# Генератор случайных вызовов

Учебный проект на `HTML`, `CSS` и `JavaScript`, который генерирует пользователю случайные квесты для саморазвития, позволяет отмечать их выполнение, ведёт историю и выдаёт награды за серии успехов.


## Назначение проекта

Система помогает пользователю получать небольшие задания на каждый день из разных категорий саморазвития:

- коммуникация;
- творчество;
- самоанализ.

Пользователь может:

- выбрать уровень сложности;
- включить нужные категории;
- сгенерировать от `1` до `3` случайных квестов;
- отметить квест как выполненный или пропущенный;
- смотреть статистику в профиле;
- получать награды за серию выполненных квестов подряд;
- просматривать историю всех созданных квестов.

## Основные возможности

### 1. Генерация ежедневных квестов

Квесты хранятся в виде массива объектов. У каждого квеста есть:

- `id` — уникальный идентификатор;
- `title` — название;
- `description` — описание;
- `difficulty` — сложность (`easy`, `medium`, `hard`);
- `category` — категория.

После нажатия кнопки `Получить квест` приложение:

1. берёт все квесты из базы;
2. фильтрует их по выбранной сложности;
3. фильтрует их по выбранным категориям;
4. случайным образом выбирает от `1` до `3` подходящих квестов;
5. показывает их на главном экране;
6. сохраняет их в историю.

### 2. Настройки сложности и категорий

В проекте есть экран `Настройки`, где пользователь может:

- выбрать сложность: `Лёгкий`, `Средний`, `Сложный`;
- отметить категории через чекбоксы.

Если категории не выбраны, приложение показывает предупреждение.

### 3. Отметка статуса квестов

Для каждого квеста доступны действия:

- `Выполнить` — статус становится `completed`;
- `Пропустить` — статус становится `skipped`.

Если квест ещё не обработан, его статус по умолчанию:

- `pending`.

После смены статуса интерфейс сразу обновляется:

- меняется цвет карточки;
- меняется бейдж статуса;
- обновляется профиль;
- обновляется история;
- данные сохраняются в `localStorage`.

### 4. Система наград

Если пользователь выполняет квесты подряд, растёт серия успехов (`currentStreak`).

Награды выдаются за серии:

- `3` квеста подряд;
- `5` квестов подряд;
- `7` квестов подряд.

Если пользователь нажимает `Пропустить`, серия обнуляется.

Награды:

- сохраняются в массиве объектов;
- отображаются в профиле;
- имеют название, описание, иконку и дату получения.

### 5. Профиль пользователя

В профиле сохраняются:

- имя;
- дата регистрации;
- текущий уровень;
- текущая серия;
- число наград.

Также отображается статистика:

- количество выполненных квестов;
- процент выполненных квестов;
- текущая серия;
- общее число наград.

### 6. История квестов

В истории выводится таблица со столбцами:

- `Название`;
- `Сложность`;
- `Категория`;
- `Статус`;
- `Дата выполнения`.

Есть фильтр по статусу:

- `all`;
- `completed`;
- `pending`;
- `skipped`.

- приветствие пользователя;

- кнопка `Получить квест`;
- список квестов на текущий день.

### Экран настроек

На экране настроек доступны:

- выбор сложности;
- выбор категорий;
- кнопка сохранения.

### Экран профиля

На экране профиля показаны:

- имя пользователя;
- дата регистрации;
- текущий уровень;
- статистика;

## Как устроен проект

Проект разбит на простые модули.

### Структура файлов

- `index.html` — основная HTML-разметка всех экранов приложения.
- `css/style.css` — стили интерфейса и адаптивная вёрстка.
- `js/app.js` — точка входа приложения, связывает все модули.
- `js/data/quests.js` — база квестов.
- `js/modules/storage.js` — загрузка и сохранение состояния в `localStorage`.
- `js/modules/profile.js` — логика профиля, уровней, статистики и наград.
- `js/modules/quest-service.js` — генерация квестов и обновление их статусов.
- `js/modules/ui.js` — работа с DOM, отрисовка экранов, уведомления.

### Роль каждого модуля

#### `quests.js`

Содержит стартовую базу квестов. Это обычный массив объектов, который можно легко показать преподавателю как пример структуры данных.

#### `storage.js`

Отвечает за:

- создание состояния по умолчанию;
- загрузку состояния из `localStorage`;
- сохранение состояния в `localStorage`;
- генерацию простых уникальных идентификаторов.

#### `profile.js`

Отвечает за:

- подсчёт выполненных квестов;
- расчёт процента выполнения;
- определение уровня пользователя;
- увеличение серии;
- сброс серии;
- добавление наград.

#### `quest-service.js`

Отвечает за:

- определение текущей даты;
- получение списка категорий;
- фильтрацию квестов;
- случайный выбор квестов;
- создание записей в истории;
- изменение статуса квеста;
- связь статуса с наградами и серией.

#### `ui.js`

Отвечает за интерфейс:

- переключение экранов;
- отрисовку карточек квестов;
- отрисовку профиля;
- отрисовку истории;
- вывод уведомлений;
- обработку нажатий кнопок и отправки форм.

#### `app.js`

Это главный файл, который:

1. загружает состояние;
2. инициализирует приложение;
3. вызывает рендер интерфейса;
4. связывает действия пользователя с логикой модулей.

## Алгоритм работы приложения

### При загрузке страницы

Приложение:

1. загружает сохранённые данные из `localStorage`;
2. если данных нет, создаёт состояние по умолчанию;
3. подставляет имя пользователя, статистику и настройки;
4. показывает квесты текущего дня, если они уже были сгенерированы;
5. отображает историю и награды.

### При нажатии `Получить квест`

Приложение:

1. берёт выбранные фильтры;
2. ищет подходящие квесты;
3. случайно выбирает от `1` до `3` элементов;
4. записывает их в раздел текущих квестов;
5. добавляет записи в историю;
6. обновляет интерфейс.

### При нажатии `Выполнить`

Приложение:

1. меняет статус квеста на `completed`;
2. увеличивает текущую серию;
3. проверяет, достигнута ли серия `3`, `5` или `7`;
4. если достигнута, добавляет награду;
5. пересчитывает статистику;
6. сохраняет всё в `localStorage`.

### При нажатии `Пропустить`

Приложение:

1. меняет статус квеста на `skipped`;
2. сбрасывает серию в `0`;
3. сохраняет изменения;
4. обновляет интерфейс.

## Какие данные хранятся в localStorage

Приложение сохраняет одно общее состояние, внутри которого есть:

- `settings` — настройки генерации и фильтра истории;
- `profile` — имя, дата регистрации, уровень, серия;
- `rewards` — список наград;
- `history` — история всех созданных квестов;
- `daily` — активные квесты на текущий день.

Это удобно, потому что все данные лежат в одном объекте, и проект легко объяснять.

## Как запустить проект

### Вариант 1. Самый простой

1. Откройте папку проекта.
2. Найдите файл `index.html`.
3. Откройте его в браузере.

Этого достаточно для работы приложения.

### Вариант 2. Через Live Server в VS Code

Если используется `VS Code`, можно:

1. открыть папку проекта;
2. нажать правой кнопкой по `index.html`;
3. выбрать `Open with Live Server`.

Этот вариант удобен для демонстрации.

## Как пользоваться приложением

1. Откройте вкладку `Профиль` и при желании задайте имя.
2. Перейдите в `Настройки`.
3. Выберите сложность.
4. Отметьте нужные категории.
5. Нажмите `Сохранить настройки`.
6. Вернитесь на `Главный экран`.
7. Нажмите `Получить квест`.
8. Отмечайте задания как `Выполнить` или `Пропустить`.
9. Смотрите изменения в `Профиле` и `Истории`.

## Проверки и обработка ошибок

В приложении предусмотрены базовые проверки:

- если база квестов пустая, выводится сообщение;
- если по выбранным фильтрам ничего не найдено, выводится сообщение;
- если не выбрана ни одна категория, выводится предупреждение;
- если имя пустое, оно не сохраняется;
- если квест уже был обработан, повторно изменить статус нельзя.
